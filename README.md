# QA Engineering Portfolio

Este repositório demonstra minha abordagem completa de Qualidade de Software ao longo de todo o ciclo de desenvolvimento de um produto.
Não me concentro apenas em automação de testes, mas sim em todo o pensamento de qualidade que deve ser valorizado, mostrando a efetividade contínua do meu trabalho.

Com experiência em QA, atuo não apenas na execução de testes, mas principalmente na **prevenção de defeitos, análise de riscos e estratégia de testes**, desde as etapas iniciais do desenvolvimento até a entrega final.

## Objetivo

Demonstrar como a qualidade é construída através de um processo estruturado, cobrindo desde a análise inicial da feature até a entrega contínua:

* Participação ativa em refinamentos de backlog
* Análise detalhada de requisitos e critérios de aceite
* Definição de estratégia de testes
* Planejamento de testes com cobertura funcional e não funcional
* Análise de risco e priorização de cenários críticos
* Design de cenários de teste manuais e BDD
* Execução de testes exploratórios
* Automação de testes end-to-end, APIs e mobile
* Monitoramento de métricas de qualidade e feedback contínuo

## Skills e Competências

**Hard Skills (Técnicas)**

* Testes Funcionais: Planejamento e execução de casos manuais e BDD
* Automação de Testes: Cypress, Cucumber, Karate DSL, Appium
* Testes de API: Validação de contratos, payloads, integração de serviços
* CI/CD e DevOps: Jenkins, GitHub Actions, integração com pipelines de teste
* Ferramentas de Gestão: Jira + Zephyr, Azure DevOps Test Plans
* Análise de Métricas: Defect leakage, coverage de testes, defect density, automation coverage
* Estratégia de Qualidade: Test strategy, risk-based testing

**Soft Skills (Relacionamento interpessoal)**

* Comunicação efetiva e objetiva: Tradução de problemas localizados, tanto para négocio quando para time técnico.
* Liderança e Mentoria: Orientação de QAs juniores e revisão de casos de teste.
* Colaboração Multidisciplinar: Trabalho próximo de desenvolvedores, PO, designers e Analistas de Négocios.
* Resolução de Conflitos: Mediação de prioridades e resolução de bugs críticos.
* Pensamento Crítico: Antecipação de riscos e identificação de gaps antes do desenvolvimento.

## Ferramentas de Automação de Testes

A automação é utilizada para garantir qualidade, confiabilidade e velocidade de feedback. Exemplos de stacks utilizadas:

* Cypress + Cucumber (JS) – Testes end-to-end em aplicações web com cenários em BDD.
* Karate DSL + Cucumber – Testes de API, validação de contratos, payloads e fluxos de integração.
* Appium + Cucumber (Java) – Testes mobile para Android e iOS, com cenários críticos em BDD.

Observação: Posso adaptar proativamente a outras ferramentas, conforme necessidade do projeto.

## Integração Contínua e Observabilidade

Experiência com ferramentas que integram testes automatizados ao fluxo de desenvolvimento e entrega contínua:

* Jenkins – Pipelines para execução automática de testes, relatórios e integração com Teams/Slack.
* GitHub / Bitbucket – Versionamento, branch management, code review e CI.
* Jira com Zephyr / Azure DevOps Test Plans – Gestão de backlog, bugs, casos de teste e integração com metodologias ágeis.


## Filosofia de Qualidade

> Testar não é apenas encontrar bugs, mas garantir que o produto entregue valor real ao usuário.
> A qualidade deve ser incorporada desde as primeiras decisões de desenvolvimento, não apenas validada no final.
